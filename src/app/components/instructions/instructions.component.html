Welcome to the ProQuo AI front-end test!

What do you need to do?

You have been tasked with completing the ProQuo AI food menu.

To accomplish this task, you have been given some components (some wired-up, some not), a mock back-end service (backend.service.ts), and the ProQuo logos (the small SVG version, and the larger PNG version).

NOTE: When the word "page" is used, it refers to the concept of a piece of paper

Implementation and Design Details

We would like to see the following:

  • We would like the menu to be contained within a 21:9 aspect ratio and should be outlined to indicate this area. This 21:9 area should be treated as a two column layout (much like a menu in any traditional restaurant that you'd open like a book)
  • A cover page for our menu.
    • This should contain the ProQuo AI logo. Remember - since this is the cover page, it should only use up one half of the available area (our menu is not yet open!)
    • It should have a button to "Open" the menu
  • All of the menu items (this is the "opened" menu)
    • Have the "pq" logo at the top of both pages
    • Display all of the menu items grouped by their type in the order: appetizer, main course, dessert
    • As display space runs out on the first page, the items should continue on to the second
  • Beneath the dessert section, we'd like to have a "Restaurant History" section that should display the data from the history API call
  • We only want all of the menu items and the history section to display when BOTH have fully loaded
  • A modal to display all information for a selected menu item
    • Along with all of the general menu item information, we would also like to display the metadata for the selected menu item
